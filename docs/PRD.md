# PRD: PyCon Korea Slack Bot

## 1. 개요

PyCon Korea 준비위원회가 사용하는 여러 협업 도구(두레이, 슬랙, 구글 캘린더, 구글 드라이브)를 슬랙 내에서 통합 관리할 수 있는 봇을 개발한다.
준비위원들이 슬랙을 떠나지 않고도 업무를 조회·생성·수정할 수 있도록 한다.

## 2. 배경 및 목적

### 문제

- 준비위원회 업무가 두레이, 구글 캘린더, 구글 드라이브 등 여러 서비스에 분산되어 있다.
- 서비스 간 컨텍스트 스위칭으로 업무 효율이 떨어진다.
- 주요 변경사항을 놓치기 쉽다.

### 목표

- 슬랙을 허브로 삼아 모든 협업 도구의 핵심 기능을 한 곳에서 수행할 수 있도록 한다.
- 중요 이벤트 발생 시 슬랙으로 실시간 알림을 제공한다.

## 3. 사용자

PyCon Korea 준비위원회 멤버 (내부 도구)

## 4. 인터랙션 방식

두 가지 방식을 모두 지원한다.

| 방식 | 사용 상황 | 예시 |
|------|-----------|------|
| 슬래시 커맨드 | 명확한 액션 수행 | `/dooray 내 업무`, `/cal 이번주`, `/notice 공지` |
| 멘션 대화 | 자연어 질의·복합 요청 | `@pyconbot 다음 회의 언제야?` |

## 5. 기능 요구사항

### 5.1 두레이 (업무 관리)

**슬래시 커맨드**: `/dooray`

| 기능 | 커맨드 / 트리거 | 설명 |
|------|-----------------|------|
| 내 업무 조회 | `/dooray me` | 나에게 할당된 진행중 업무 목록 |
| 업무 생성 | `/dooray create` | 모달 폼으로 업무 생성 (프로젝트, 제목, 담당자, 우선순위) |
| 업무 상태 변경 | `/dooray done <업무ID>` | 업무를 완료 처리 |
| 업무 코멘트 | `/dooray comment <업무ID> <내용>` | 업무에 코멘트 추가 |
| 알림 | 웹훅 수신 | 업무 할당·상태변경·코멘트 시 슬랙 채널에 알림 |

**API 연동**: Dooray API (`https://api.dooray.com`) — Personal Authentication Token 인증

### 5.2 구글 캘린더 (일정 관리)

**슬래시 커맨드**: `/cal`

| 기능 | 커맨드 / 트리거 | 설명 |
|------|-----------------|------|
| 오늘 일정 | `/cal today` | 오늘의 일정 목록 |
| 이번주 일정 | `/cal week` | 이번주 일정 목록 |
| 일정 생성 | `/cal create` | 모달 폼으로 일정 생성 (제목, 날짜/시간, 참석자) |
| 일정 수정 | `/cal edit <일정ID>` | 모달 폼으로 일정 수정 |
| 데일리 리마인더 | 자동 (매일 오전) | 당일 일정을 지정 채널에 자동 포스팅 |

**API 연동**: Google Calendar API — OAuth 2.0 (Service Account)

### 5.3 구글 드라이브 (파일 관리)

**슬래시 커맨드**: `/drive`

| 기능 | 커맨드 / 트리거 | 설명 |
|------|-----------------|------|
| 파일 검색 | `/drive search <키워드>` | 공유 드라이브에서 파일 검색 |
| 최근 파일 | `/drive recent` | 최근 수정된 파일 목록 |
| 파일 공유 링크 | `/drive link <파일ID>` | 파일의 공유 링크 생성 |
| 파일 업로드 | `/drive upload` | 슬랙에 첨부된 파일을 지정 폴더에 업로드 |
| 알림 | 웹훅 수신 | 공유 드라이브에 파일 추가·수정 시 알림 |

**API 연동**: Google Drive API — OAuth 2.0 (Service Account)

### 5.4 공지 관리

**슬래시 커맨드**: `/notice`

#### 5.4.1 공지 메시지

| 기능 | 커맨드 / 트리거 | 설명 |
|------|-----------------|------|
| 공지 작성 | `/notice create` | 모달 폼으로 공지 작성 (제목, 내용, 대상 채널) |
| 공지 목록 | `/notice list` | 게시된 공지 목록 조회 |
| 읽음 확인 | `/notice status <공지ID>` | 공지별 멤버 읽음/미읽음 현황 조회 |
| 리마인드 | `/notice remind <공지ID>` | 미읽음 멤버에게 리마인드 DM 발송 |

**동작 방식**:
- 공지 메시지에 "확인" 버튼을 포함하여 게시한다.
- 멤버가 "확인" 버튼을 누르면 읽음 처리된다.
- 공지 작성자와 관리자는 읽음/미읽음 현황을 조회할 수 있다.

#### 5.4.2 정기 회의 공지

| 기능 | 커맨드 / 트리거 | 설명 |
|------|-----------------|------|
| 회의 공지 작성 | `/notice meeting` | 모달 폼으로 회의 공지 작성 (제목, 일시, 장소, 안건) |
| 참석 여부 응답 | 버튼 인터랙션 | 온라인 참석 / 오프라인 참석 / 미참석 중 선택 |
| 참석 현황 조회 | `/notice meeting status <공지ID>` | 회의별 참석 현황 (온라인·오프라인·미참석·미응답) 조회 |
| 참석 리마인드 | `/notice meeting remind <공지ID>` | 미응답 멤버에게 참석 여부 확인 DM 발송 |

**동작 방식**:
- 회의 공지 메시지에 "온라인 참석", "오프라인 참석", "미참석" 세 가지 버튼을 포함하여 게시한다.
- 멤버가 버튼을 누르면 참석 여부가 기록되고, 언제든 다시 눌러 변경할 수 있다.
- 회의 공지에는 일시, 장소(온/오프라인), 안건이 포함된다.
- 공지 작성자와 관리자는 참석 현황을 카테고리별(온라인·오프라인·미참석·미응답)로 조회할 수 있다.

### 5.5 멘션 대화

`@pyconbot`으로 멘션하면 자연어로 위 기능들을 수행할 수 있다.

| 예시 질의 | 동작 |
|-----------|------|
| `@pyconbot 내 두레이 업무 보여줘` | 두레이 내 업무 조회 |
| `@pyconbot 내일 오후 2시에 "스폰서 미팅" 일정 만들어줘` | 구글 캘린더 일정 생성 |
| `@pyconbot 예산 관련 파일 찾아줘` | 구글 드라이브 검색 |
| `@pyconbot 이번주 회의 참석 현황 알려줘` | 회의 공지 참석 현황 조회 |

## 6. 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 인증 | 각 외부 서비스의 인증 정보는 환경 변수로 관리한다 |
| 권한 | 슬랙 워크스페이스 멤버만 사용 가능하다 |
| 응답 시간 | 슬래시 커맨드 응답은 3초 이내에 ack하고, 결과는 비동기로 전달한다 |
| 에러 처리 | 외부 API 실패 시 사용자에게 명확한 에러 메시지를 반환한다 |
| 로깅 | 모든 커맨드 실행과 API 호출을 로깅한다 |
| 배포 | Socket Mode(개발)과 HTTP Mode(운영) 모두 지원한다 |
